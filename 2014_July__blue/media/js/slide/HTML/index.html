    Setting up your DDSlider

    Ensure you have all files needed

    Very often people can't get their slider to work before there is a file missing or wrong path to a file. Please ensure that you have all files listed below and tha all of them have the correct path.

    • jquery.min.css: jQuery Framework file. Host this locally or hotlink it from google.
    • jquery.easing.1.3.js: jQuery easing plugin javascript file. This file is needed if you want a different ease than the standard jquery 'swing'.
    • jquery.DDSlider.min.js: DDSlider jQuery Plugin javascript file. Contains all functions and events.
    • DDSlider.css: All modifications you might do will be in this file. Slider does NOT work without this.

    This files must be ALL within the <head> of your file. Make sure to put them in the right order.

    Also, don't forget to add the DDSlider folder in your images folder. Arrows and Selectors are in that file.

    <head> <!-- Slider Configurations - VERY IMPORTANT --> <link href="DDSlider.css" rel="stylesheet" type="text/css" /> <!-- jQuery framework --> <script type="text/javascript" src="js/jquery.min.js"></script> <!-- jQuery easing plugin - VERY IMPORTANT --> <script type="text/javascript" src="js/jquery.easing.1.3.js"></script> <!-- DDSlider jQuery Plugin - VERY IMPORTANT --> <script type="text/javascript" src="js/jquery.DDSlider.min.js"></script> </head>

    Create the slider structure

    The HTML basic structure of the slider is very important. You will need to defined the slider's width and height in the DDSlider.css file in order for this to work. The main parent of the slider has to be a <ul> that you wish.

    The slider is mainly composed by 1 container:

    • ul#yourSliderId: This is the mother container of your slider, all <li> tags in it will become items of the slider.
    <ul id="yourSliderId"> <li> [...] </li> </ul>

    IMPORTANT: Do not forget to adjust your slider's height and width in the DDSlider.css file. And make sure your mother container has the same ID as the one in the DDSlider.css file.

    Trigger DDSlider:

    Now it is time to trigger DDSlider so jQuery actually executes it. Add the following code to your header: (further on you'll learn how to configure your DDSlider)

    <head> <script type="text/javascript"> //triggers the slider $(document).ready(function() { $('#yourSliderId').DDSlider(); }); </script> </head>

     

     

    Adding Content and Defining Transitions

    Adding sliders

    To add new items to DDSlider is fairly simple. Every <li></li> item withing the mother container will count as a slider item, therefore adding a selector.

    <ul id="yourSliderId"> <li> [...] </li> <!-- One Slider Item --> </ul>

    Adding Sliders with Inline Content

    In order to add Sliders with inline content (this means actual HTML instead of lame static images) you should follow the same step as above with a few additional things.

    We highly recommend you to define a background color for your <li></li> tag to avoid weird transitions. Even with images as background, we advise to add a solid color as well. Follow the example below:

    <ul id="yourSliderId"> <li style="background: #ffffff;"> [...] </li> </ul>

    jQuery will now add this background option in the transition, so the transition is solid and smooth.

    Defining transitions

    You also have the possibility to define specific transitions for each slider. To do that you must first choose one of the transitions below:

    • random: Will randomly play one of the transitions below.
    • fading: Normal standard fading transition.
    • barTop: Bars animation coming from top left.
    • barBottom: Bars animation coming from bottom right.
    • barFading: Bars animation fading from left to right - curtain style.
    • barRandom: Bars animation fading at random.
    • square: Square animation fading from left to right.
    • squareMoving: Square animation where squares fade and move from left to right.
    • squareRandom: Square animation fading squares at random.
    • squareOut: Square animation going out from top left.
    • squareOutMoving: Square animation going out and moving from top left.
    • rowInterlaced: Rows animation coming from both left and right.

    Now that you have chose your transition, all you need to do is add a class="" property to your slider's <li></li> tag, with the name of your transition.

    <ul id="yourSliderId"> <li class="squareRandom"> [...] </li> </ul>

    Please note that if written wrong, the slider will play the default transition.

     

     

    Inline Content

    When we talk about inline content, it includes any HTML that you might wanna include in your slides. Buttons, forms, lists, text etc... However, if you don't know HTML and CSS I highly recommend you to learn the basics in order to take full advantage of the inline content feature.

    The plugin deals with wahtever is inside the <li> tag. Meaning that whatever you have in there, the plugin will clone your content and animate it. So in order to use the feature at its best we need to do the following:

    • 1) We need to create a wrapping container, that fills our entire slide area with a background color or image, to avoid transparent faulty transitions
    • 2) We need to set the proper width ad height of this container, so it actually fills the entire area.
    • 3) We will probably be adding padding to this container, so our inline content is not at the edge of the slide, resulting in ugly badly done content.
    • 4) Finally add our inline content.

     

    Adding our wrapping container:

    Let's start by adding a wrapping container with a background so we don't get transparent transitions. For this example I'll be using our default width and height values - 960x300.

    <li> <div style="background: #ffffff url(your background image url) repeat-x top left"> [...] </div> </li>

    With our DIV container wrapping all of our content, we don't have to bother about transparent transitions anymore. But if you add just a little of text, it won't matter much, because or div wrapping container doesn't have and width and height set so it'll automatically assign the width and height of our content (which can be as little as line height). So let's also assign a width and height:

    <li> <div style=" background: #ffffff url(your background image url) repeat-x top left; width: 960px; height: 300px; "> [...] </div> </li>

    Now we've just added width and height to our DIV container, this exapands our box, and now it fills the entire LI! But hold on, my content is right on the edge of my slider, how can I add some margin to it?

    Simple, we'll add padding to our DIV container. Adding padding can be quite confusing if you are not used to HTML & CSS. Whenever you add padding to a container that has a width and height defined, the padding is added to the width and height. For example, if you have a 1000px long DIV container, and adding 100px of padding on the left, your DIV container will actually be 1100px wide and not 1000px. Said so, whenever you add padding to the left or right, you'll need to subtract this value from the width, and whenever you add padding on the top or bottom, you'll need to subtract the padding from the height value.

    In the next example I'll be adding 50px padding on each side of my DIV container, pay attention how my width and height will be reduced.

    <li> <div style=" background: #ffffff url(your background image url) repeat-x top left; width: 860px; /* (which is actually 960px - 50px - 50px) */ height: 200px; /* (which is actually 300px - 50px - 50px) */ padding-top: 50px; padding-right: 50px; padding-bottom: 50px; padding-left- 50px; "> <h2>Now I can add Inline Content!</h2> </div> </li>

    From here on, you are free to insert buttons, links, paragraphs, lists and much more. I won't get into details of adding buttons and etc because the point of this documentation is to just explain the inline content function and not to teach HTML and CSS.

     

     

    Customize your slider

    In order to customize your slider, you must edit the DDSlider.css file. In there you will find a full description of each of the parts of this slider. Do not forget to edit your slider's width and height in the css file.

    Other customizations like arrows and selectors are in the style.css file, however, the slider works fine without it.

     

     

    Configure your DDSlider

    DDSlider gives you the option to fully customize your slider. Below you'll find a list of all options available and how to set them.

    Remember when we triggered DDSlider in our <head> tag? Well to configure our DDSlider we'll have to go back up again. Below there is a full list of all options available and its default values

    <script type="text/javascript"> $(document).ready(function() { $('#yourSliderId').DDSlider({ trans: 'random', delay: 50, ease: 'swing', waitTime: 5000, duration: 500, stopSlide: 1, bars: 15, columns: 9, rows: 3, selector: null, arrowNext: null, arrowPrev: null }); }); </script>
    • trans: Choose here your default transition. If no transition is defined in the slider item, this is the transition to be played.
      Default: random
    • delay: Delay between each block of slider is played in miliseconds. ie: the first block starts animating at 0, second starts at x, third starts at 3x and so on.
      Default: 50
    • ease: Type of easing that the animation is going to have. If you have the jQuery easing plugin in your website, you may choose one from there. Remember to put in between apostrophes or quotes.
      Default: 'swing'
    • waitTime: Delay time between each slider in miliseconds. This is the wait time for the next slide to be played automatically.
      Default: 5000
    • duration: Duration of animations in miliseconds. The lower this number is, the faster the animations are gonna be played.
      Default: 500
    • stopSlide: This options does not have a function. Will be available in future updates.
      Default: 1
    • bars: Number of 'slices' to be played in in the 'bars' transitions. Keep this number dividable by the width.
      Default: 15
    • columns: Number of columns to be played in the 'square' transitions.
      Default: 9
    • rows: Number of Rows to be played in the 'square' transitions.
      Default: 3
    • selector: Sets a given selector as the slider selector. The little circles over the slider.
      Default: null
    • arrowNext: Sets a given object so when this is clicked the next slider is called. Works as a next button.
      Default: null
    • arrowPrev: Sets a given object so when this is clicked the previous slider is called. Works as a previous button.
      Default: null

    Please note that your selector container HAS TO BE a <ul></ul> container. Selectors are added as empty <li></li> tags.

    Current selectors are given the class '.current'. You might wanna change the image of the current selector so it differs from the rest.
